# Final Week Project Options

This folder contains final-week project options for the training program.

Each team should pick **one** project and use its specification document as the source of detailed requirements.

## How to Choose

Pick a project based on the type of problem your team wants to solve:

- **Payment Processing**: Build a workflow-driven API for payment lifecycle tracking.
- **Portfolio Manager**: Build an API and UI for managing and visualizing portfolio holdings.
- **Transaction Monitoring**: Build a rules-based monitoring system with alert lifecycle management.

## Project Summaries

### 1) Payment Processing System

- **Focus**: Payment lifecycle and status transitions.
- **Typical Deliverables**: REST API, persistence layer, status history/audit trail, optional UI.
- **Good Fit For Teams Who Want**: Strong backend workflow logic, state management, and error-handling design.
- **Specification**: [payment_processing.md](payment_processing.md)

### 2) Portfolio Manager

- **Focus**: Portfolio data modeling, performance views, and user-facing insights.
- **Typical Deliverables**: REST API, persistence layer, portfolio UI, optional market data integration.
- **Good Fit For Teams Who Want**: A balanced backend + frontend build with opportunities for data visualization.
- **Specification**: [portfolio_manager.md](portfolio_manager.md)

### 3) Transaction Monitoring & Alerts Dashboard

- **Focus**: Rule evaluation, event detection, and alert workflows.
- **Typical Deliverables**: REST API, rules engine basics, transaction storage, alerts dashboard/UI.
- **Good Fit For Teams Who Want**: Detection logic, operational workflows, and scalable processing patterns.
- **Specification**: [transaction_monitoring.md](transaction_monitoring.md)

## Common Expectations (All Projects)

- Start with a **minimal viable implementation** and expand incrementally.
- Use the training stack for API, database, and frontend.
- Work as a team with clear task ownership and regular check-ins.
- Use source control practices (branches, pull requests, and clear commits).
- Prepare a short final presentation/demo explaining decisions, tradeoffs, and next steps.
